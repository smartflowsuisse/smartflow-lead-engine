## 10. What not to change

Client should not change without asking SmartFlow:

- column names;
- sheet structure;
- form field names;
- Make scenario modules;
- connected accounts;
- AI prompt;
- notification recipient;
- domain/DNS settings;
- access permissions.

Reason:

Small structural changes can break the workflow.

---

## 11. What to do if something breaks

Basic troubleshooting path:

1. Stop changing settings.
2. Take a screenshot of the issue.
3. Note the date/time.
4. Note what input was used.
5. Check whether the source form/sheet still works.
6. Send the issue to SmartFlow.
7. Do not delete rows, modules, or access before review.

Emergency note:

If client data or external sending is affected, pause the workflow until reviewed.
