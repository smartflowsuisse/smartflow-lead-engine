# First Client Call Sheet

## Purpose

Short internal call sheet for a first client conversation or demo.

This document helps keep the call simple, safe, and focused on one clear next step.

## Status

Client-facing: No  
Manual review required before client-facing use: Yes

## Call objective

The goal is not to sell a large automation project immediately.

The goal is to:

- understand how the client currently receives and follows requests;
- show a simple controlled demo;
- explain that SmartFlow keeps human review in the process;
- identify one useful, low-risk next step.

## 60-second explanation in French

SmartFlow Suisse aide les PME à mieux gérer leurs demandes et leurs suivis.

Dans cette démonstration, une demande arrive via un formulaire, elle est ajoutée dans un mini CRM Google Sheets, puis l’IA prépare un résumé interne.

Aucun email n’est envoyé automatiquement.

L’objectif est d’aider l’équipe à comprendre plus vite la demande, à prioriser le suivi et à préparer une réponse de manière plus efficace.

L’humain garde toujours le contrôle.

## What to show

Show only the controlled demo flow:

1. Demo request.
2. CRM row.
3. AI internal summary.
4. Manual next step.
5. Safety: no automatic external email.

## Questions to ask

Ask simple process questions:

- Comment vos demandes arrivent aujourd’hui ?
- Où sont-elles suivies ?
- Qui répond au client ?
- Quelles tâches sont répétitives ?
- Où perdez-vous le plus de temps ?
- Quel suivi est le plus difficile à contrôler ?

## What not to promise

Do not promise:

- full automation before scope review;
- automatic external email sending;
- production deployment during the first call;
- CRM migration without analysis;
- database or API changes without technical review;
- fixed final pricing without scope confirmation.

## Safe next step options

Offer one clear next step:

1. Short discovery call.
2. Review of current request-tracking process.
3. One-page mini-audit.
4. Limited pilot workflow.
5. Written scope proposal.

Recommended first next step:

- review the current workflow first, then propose one small improvement that can be implemented without disrupting the existing process.

## Safety rules

- Documentation only.
- Internal use only.
- No production changes.
- No Make schedule changes.
- No automatic external email sending.
- No database or API changes.
- Manual review required before any client-facing wording.
