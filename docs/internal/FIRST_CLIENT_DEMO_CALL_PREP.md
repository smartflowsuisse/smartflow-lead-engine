## Demo Goal

Show a simple and safe preview of how SmartFlow Suisse can help a Swiss SME manage leads, follow-ups, and client requests without introducing risky automation too early.

The goal is not to sell a large system immediately.

The goal is to qualify interest and propose a small controlled pilot.

## Demo Flow

### 1. Start With The Client Problem

Explain that many small and mid-sized companies lose time because client requests arrive through different channels and are handled manually.

Typical issues:

- missed follow-ups
- unclear lead priority
- repeated manual admin work
- no simple overview of current opportunities
- no structured next action

### 2. Show The Homepage Offer

Show the Lead & Follow-up System section.

Explain that this is not a generic AI tool.

It is a practical workflow for:

- capturing incoming requests
- summarizing them
- prioritizing them
- preparing follow-up
- keeping human approval before action

### 3. Show Client Hub Preview

Open the Client Hub preview.

Explain that this is a safe preview layer, not a live production CRM.

Show:

- lead overview
- AI summary
- priority
- next action
- manual review concept

### 4. Show Convert To Client Preview

Explain how a lead can move from request to potential client.

Important point:

No real client is created automatically.

The flow is designed around human review and approval.

### 5. Show Make Demo Story

Open the demo Google Sheet and Make scenario.

Explain the safe demo chain:

Form / Sheet
→ Make
→ AI summary
→ CRM Demo Sheet

Important:

- automation is not enabled for production
- Gmail sending is not enabled
- no external email is sent
- manual review remains required

### 6. Explain Safety Boundaries

Clearly say:

SmartFlow does not start with uncontrolled automation.

The first version should be small, reviewed, and reversible.

Safety principles:

- no automatic client creation
- no automatic external emails
- no mass outreach
- no production automation without approval
- human review before client-facing actions
- rollback and audit thinking from the start

### 7. Move Toward Pilot

Explain that the recommended first step is a small pilot.

Pilot scope:

1 request type
→ 1 intake channel
→ 1 lead workflow
→ 1 follow-up process
→ 1 approval point
→ 1 review cycle

Estimated range:

CHF 1,500 – 4,500

Estimated timeline:

7–21 days

## Questions To Ask The Client

Ask these questions during or after the demo:

1. Where do most client requests arrive today?
2. Who currently handles new inquiries?
3. How do you decide which lead is urgent?
4. How often are follow-ups missed or delayed?
5. Do you already use a CRM, spreadsheet, or inbox-based process?
6. What is the most repetitive admin step after a new request arrives?
7. Would you prefer a simple internal dashboard or email-based workflow first?
8. Who should approve follow-up messages before they are sent?
9. What would make a pilot useful in the first 2–3 weeks?
10. What should not be automated under any circumstances?

- full CRM replacement immediately
- guaranteed revenue increase

- instant integration with every tool

- fixed price without scoping
- same-day full implementation
