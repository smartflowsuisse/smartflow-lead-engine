# First Client Demo Sheet Draft

## Purpose

This document prepares a short one-page demo sheet for a first client conversation.

It is an internal draft for manual review before any client-facing use.

## Status

Client-facing: No  
Manual review required: Yes

## Demo context

Use this sheet when presenting the SmartFlow follow-up and request-tracking demo to a small business client.

Primary example:

- sector: construction, renovation, menuiserie, home services, or similar local service business;
- problem: client requests arrive through several channels and are difficult to track manually;
- proposed demo: request intake → CRM row → AI-assisted internal summary → manually reviewed draft reply.

## Client problem

The client may have:

- scattered requests from phone, email, forms, WhatsApp, or referrals;
- no simple overview of open leads;
- slow follow-up after a first request;
- repeated manual copy-paste between tools;
- lost opportunities because no clear next step is tracked.

## SmartFlow demo message

SmartFlow Suisse can show a controlled workflow that helps:

- capture incoming requests in one structured place;
- generate an internal AI-assisted summary;
- prepare a draft reply for manual review;
- keep follow-up status visible;
- reduce manual admin work without removing human control.

## What the demo shows

The demo can show:

1. A sample client request.
2. A CRM row created from the request.
3. An AI-assisted internal summary.
4. A draft reply prepared for review.
5. A clear follow-up status.
6. A manual next-step decision.

## What the demo does not do

The demo does not:

- send external emails automatically;
- enable Make scheduling without review;
- change production systems;
- change database or API logic;
- replace human validation;
- promise a final client system before scope confirmation.

## Safe explanation for the client

Suggested wording for manual adaptation:

SmartFlow Suisse can help you centralize incoming requests, structure follow-up, and prepare AI-assisted summaries or draft replies. The goal is not to automate everything blindly, but to reduce repetitive admin work while keeping human review before any client-facing action.

## Proposed next step

After the demo, propose one of these next steps:

- short discovery call;
- review of current request-tracking process;
- simple mini-audit;
- limited pilot workflow;
- written scope proposal.

## Internal safety rules

- Documentation only.
- No production changes.
- No Make schedule changes.
- No automatic external email sending.
- No database or API changes.
- Manual review required before client-facing use.
