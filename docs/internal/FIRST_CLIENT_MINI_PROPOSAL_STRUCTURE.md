# SmartFlow First Client Mini Proposal Structure

## Purpose

This document defines a short internal structure for preparing a first-client mini proposal after a reply, discovery call, demo, or one-page audit.

The goal is to propose one clear, safe, limited next step.

This is not a full commercial proposal.

## Core principle

A mini proposal should be:

- short;
- specific;
- based on the client's real workflow;
- limited in scope;
- easy to understand;
- safe to execute;
- manually reviewed before sending.

Do not use a mini proposal to push a large project before the workflow is understood.

## When to use this document

Use this structure after:

- a positive client reply;
- a discovery call;
- a first demo;
- a one-page audit;
- a clear workflow problem has been identified.

Do not use it for mass outreach.

## Recommended structure

### 1. Client context

Record:

- company name;
- contact person;
- language;
- date;
- source of conversation;
- current workflow discussed;
- main pain point.

### 2. Problem observed

Describe the problem in simple terms.

Good examples:

- requests arrive from several channels;
- follow-up is manual and easy to miss;
- quote requests are not tracked clearly;
- website form submissions are not connected to a follow-up process;
- internal replies take time to prepare.

Avoid:

- vague AI language;
- blaming the client;
- promising transformation before analysis.

### 3. Proposed quick win

Choose one small improvement.

Examples:

- simple CRM tracking sheet;
- form-to-CRM workflow;
- internal notification workflow;
- AI summary for manual review;
- reply draft prepared for human approval;
- follow-up reminder process;
- contact-form cleanup.

The quick win must be small enough to explain clearly.

### 4. Scope included

List what is included.

Example:

- review current intake workflow;
- define fields for tracking;
- create or clean a simple CRM structure;
- prepare one automation draft;
- test with demo or low-risk data;
- document manual review steps;
- hand over a simple operating guide.

### 5. Scope excluded

List what is not included.

Example:

- no automatic external email sending;
- no mass outreach;
- no full CRM migration;
- no production changes without QA;
- no sensitive data processing unless approved;
- no complex multi-system rollout;
- no fixed savings guarantee.

### 6. Safety and approval

State the safety rules clearly.

Required points:

- human review before any external message;
- demo or low-risk data preferred;
- no automatic sending;
- no production changes without QA;
- account and ownership verification before integrations;
- sensitive data must be excluded unless necessary and approved.

### 7. Timeline

Use cautious language.

Good wording:

- estimated timeline;
- depends on access and workflow clarity;
- first setup can be limited;
- implementation starts only after scope approval.

Avoid:

- guaranteed completion date without checking access;
- unrealistic speed;
- promises before account verification.

### 8. Price or budget

Use a careful format.

Options:

- fixed price after scope validation;
- small audit first;
- quick-win package;
- phased approach.

Do not invent a final price if the workflow is not understood.

Internal known offer ranges:

- workflow audit: 590–900 CHF;
- automation quick win: 1,200–5,000 CHF+;
- foundation web + AI: 3,500–7,500 CHF;
- monthly care: from 149 CHF/month.

Use ranges only when appropriate.

### 9. Client next step

Offer one clear next step.

Examples:

- approve a short audit;
- approve a small quick win;
- send access information;
- schedule a 20–30 minute call;
- confirm the workflow to improve;
- no action now.

Do not offer too many choices.

## French client-facing skeleton

Subject:

Proposition de prochaine étape — [workflow/client]

Email:

Bonjour [Nom],

Merci pour votre retour et pour les informations partagées.

D’après ce que nous avons compris, le point principal à améliorer serait :

- [problème principal]

Pour commencer de manière prudente, nous proposons une première étape limitée :

- [quick win proposé]

Cette première étape inclurait :

- [élément 1]
- [élément 2]
- [élément 3]

Elle n’inclurait pas :

- l’envoi automatique d’emails externes;
- une migration complète;
- des changements en production sans validation;
- l’utilisation de données sensibles sans accord explicite.

L’objectif est de tester une amélioration concrète avec un périmètre clair, puis de décider ensuite si une étape plus large est pertinente.

La prochaine étape serait de [next step].

Meilleures salutations,

SmartFlow Suisse

## Do not say

Avoid:

- everything will be automated;
- AI will replace your team;
- we can connect everything immediately;
- this will save a fixed number of hours;
- automatic emails will be sent without review;
- the setup is risk-free;
- no work is required from the client;
- the result is guaranteed before testing.

## Internal decision logic

If the workflow is unclear:

- propose a paid workflow audit.

If the workflow is clear and low-risk:

- propose a small automation quick win.

If the website/contact path is weak:

- propose contact-form or intake cleanup.

If the client is interested but cautious:

- propose a demo or small audit.

If the client has sensitive data concerns:

- clarify data boundaries first.

If the client is not ready:

- do not push.

## Risk checklist

Before sending a mini proposal, check:

- company name is correct;
- contact person is correct;
- language is correct;
- problem is based on real information;
- no false claim is included;
- no fixed savings are promised without analysis;
- no automatic sending is proposed;
- no production change is proposed without QA;
- no sensitive data is requested unnecessarily;
- next step is clear.

## Safety rules

- No automatic external email sending.
- No mass outreach.
- Human approval required.
- No sensitive data exposure.
- No unverified recipient addresses.
- No overpromising.
- No fixed savings without analysis.
- No production changes without QA.
- Do not use real client data in AI tools unless necessary and approved.
- Client-facing wording must be reviewed manually before sending.
