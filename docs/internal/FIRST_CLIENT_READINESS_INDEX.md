# SmartFlow First Client Readiness Index

## Purpose

This index explains which internal SmartFlow documents to use when preparing for the first client.

The goal is to avoid scattered notes and keep the workflow clear, safe, and repeatable.

## Current principle

SmartFlow Lead Engine is an internal tool.

Current priority:

- first-client readiness;
- safe demos;
- clear offers;
- manual outreach control;
- data safety;
- reply handling;
- practical client workflow preparation.

Not current priority:

- public SaaS launch;
- automatic mass outreach;
- automatic external email sending;
- uncontrolled production changes;
- broad feature expansion without client validation.

## Default workflow for preparing a first client

### Step 1 — Select a lead or demo client

Check:

- company name;
- industry;
- website;
- language;
- contact data;
- notes;
- likely business pain.

### Step 2 — Review safety

Before any external action, verify:

- no automatic email sending;
- no mass outreach;
- no sensitive data exposure;
- no production changes;
- human review is required.

### Step 3 — Prepare discovery

Use:

- discovery call questions;
- client workflow templates;
- first offer draft.

Goal:

- understand the client workflow before proposing a solution.

### Step 4 — Prepare demo

Use:

- client demo checklist;
- Make demo workflow;
- Google Sheets CRM demo structure.

Goal:

- show a simple, safe, human-supervised workflow.

### Step 5 — Decide next step

Recommended next steps:

- paid workflow audit;
- small automation quick win;
- CRM cleanup;
- website/contact-form improvement;
- follow-up proposal.

Avoid:

- large project proposal before discovery;
- automatic outreach;
- unverified promises;
- production changes during demo.

## Safety rules

### No automatic sending

Do not enable automatic external email sending without explicit approval.

### No mass outreach

Do not use Lead Engine or Make demo for bulk outreach.

### Human approval required

A human must review and approve any external message.

### Account verification required

Before changing Google, Make, Gmail, or Drive connections, verify the account and ownership.

Preferred account:

- smartflowsuisse@gmail.com

### No production changes without QA

Do not change production systems without checking the current state, expected impact, and rollback risk.

### No sensitive data exposure

Use fake or low-risk demo data by default.

Do not show confidential client data unless explicitly approved and necessary.
