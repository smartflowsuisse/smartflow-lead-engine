# SmartFlow Suisse — Reply Handling Process Draft

## 1. Purpose

This document defines how SmartFlow Suisse handles replies after manual outreach or client contact.

Goal:

- avoid confusion after a reply;
- keep CRM status accurate;
- choose the right next action;
- avoid overpromising;
- prepare for a call, mini-audit, or proposal.

## 2. Main rule

A reply does not automatically mean a sale.

Every reply must be reviewed manually.

Do not change a lead to Won, Proposal, or Meeting unless that event really happened.

## 3. CRM status logic

Recommended flow:

New / Analyzed → Contacted → Replied → Meeting → Proposal → Won / Lost

Use statuses carefully:

- Contacted: a real message was sent.
- Replied: the company replied.
- Meeting: a call or meeting is scheduled or completed.
- Proposal: a specific offer/proposal was sent.
- Won: the client accepted a paid offer.
- Lost: the client declined or is not a fit.

## 4. Reply types

### Type A — Interested

Examples:

- “Yes, send more information.”
- “This could be useful.”
- “Can you explain what you do?”

Next action:

1. Mark CRM as Replied.
2. Add notes with reply summary.
3. Send short answer.
4. Offer a short audit call or ask 2–3 discovery questions.
5. Create follow-up task.

### Type B — Asks for price

Examples:

- “How much does it cost?”
- “What are your rates?”

Next action:

1. Mark CRM as Replied.
2. Do not send complex pricing immediately.
3. Explain that price depends on workflow scope.
4. Give simple range if appropriate.
5. Suggest a short audit to identify one practical quick win.

Safe answer:

Le prix dépend du workflow à automatiser. Pour un premier quick win simple, nous travaillons généralement sur une base claire après un court audit du processus actuel.

### Type C — Wants a call

Examples:

- “Can we talk?”
- “Are you available this week?”

Next action:

1. Mark CRM as Replied.
2. Create calendar task.
3. Prepare discovery call checklist.
4. Send 2–3 possible time slots.
5. Do not send proposal before understanding the workflow.

### Type D — Asks for details

Examples:

- “What exactly do you automate?”
- “Can you give examples?”

Next action:

1. Mark CRM as Replied.
2. Send 2 or 3 concrete examples.
3. Avoid vague AI language.
4. Offer a simple audit.

Examples to mention:

- contact form → Google Sheets CRM;
- request → follow-up task;
- email/PDF → structured summary;
- AI draft reply for human review.

### Type E — Not interested

Examples:

- “Not interested.”
- “We do not need this.”

Next action:

1. Mark CRM as Lost.
2. Add short note.
3. Do not follow up unless they invite future contact.
4. Keep tone polite.

### Type F — Unclear reply

Examples:

- vague response;
- automatic reply;
- out-of-office;
- forwarded internally.

Next action:

1. Do not mark as Won or Meeting.
2. Add note.
3. If human reply exists, mark Replied.
4. If automatic reply only, keep Contacted.
5. Create review task if needed.

### Type G — No response

Next action:

1. Do not send new outreach automatically.
2. Wait until follow-up date.
3. Review whether follow-up is appropriate.
4. If follow-up is sent, it must be manual and reviewed.
5. If not appropriate, mark Lost or keep Contacted with note.

## 5. Internal note format

When a reply arrives, add a note:

Reply received.
Date: YYYY-MM-DD
Channel: Email / Phone / Form / Other
Summary:
Next action:
CRM status changed from X to Y.

## 6. Follow-up task rules

Create a task only when there is a real next action.

Good task titles:

- Review reply and prepare answer
- Prepare discovery call questions
- Send requested examples
- Prepare mini-audit
- Review whether follow-up is appropriate
- Check reply after manual outreach

Avoid risky task titles:

- Send automatic follow-up
- Push client again
- Mark as Won
- Send proposal without review

## 7. First reply response structure

A safe reply should include:

1. Thank you.
2. Short clarification.
3. 1 or 2 concrete examples.
4. One simple next step.
5. No pressure.

Example structure in French:

Bonjour,

Merci pour votre retour.

L’idée est de commencer simplement: regarder comment vous gérez aujourd’hui les demandes entrantes, les relances et les tâches administratives répétitives, puis identifier 1 ou 2 automatisations utiles sans changer vos outils existants.

Par exemple:
- formulaire de contact → tableau de suivi;
- demande client → tâche de relance;
- email ou PDF → résumé structuré pour validation humaine.

Si cela vous convient, je peux vous proposer un court audit gratuit de votre workflow actuel.

Cordialement,
SmartFlow Suisse

## 8. Discovery call preparation

Before a call, prepare:

- company website;
- current CRM status;
- last message;
- suspected pain;
- 3 workflow questions;
- possible quick win;
- safety boundaries;
- next step after call.

Core questions:

1. Où arrivent vos demandes clients aujourd’hui?
2. Comment suivez-vous les relances?
3. Quelle tâche administrative vous fait perdre le plus de temps chaque semaine?
4. Utilisez-vous déjà un tableau, un CRM ou seulement l’email?
5. Qui valide les réponses envoyées aux clients?

## 9. When to prepare a mini-audit

Prepare a mini-audit if:

- client shows interest;
- workflow pain is visible;
- public website/contact process can be reviewed;
- there is a realistic quick win.

Do not prepare a full proposal before:

- understanding the workflow;
- confirming the pain;
- knowing who decides;
- estimating implementation scope.

## 10. Proposal readiness

Send a proposal only when:

- client problem is clear;
- one quick win is identified;
- scope is limited;
- required tools are known;
- safety rules are clear;
- price range is justified;
- next step is practical.

## 11. Safety reminders

Do not:

- promise guaranteed sales;
- say AI will handle everything;
- request all client data;
- automate sending without approval;
- change client tools without confirmation;
- move to Won without real acceptance.
