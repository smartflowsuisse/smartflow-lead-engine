# SmartFlow Suisse — Internal Client Readiness Kit

Status: internal only  
Project: SmartFlow Lead Engine  
Purpose: prepare SmartFlow Suisse for first real clients without public SaaS launch.

## Core rule

SmartFlow Lead Engine is an internal tool.

Do not:
- publish it as SaaS;
- give client access;
- connect payment/subscription;
- send automated emails;
- mark leads as Contacted before real manual sending;
- commit local database files;
- deploy externally.

Use it for:
- lead import;
- website analysis;
- CRM tracking;
- French outreach draft generation;
- manual review;
- manual sending;
- follow-up planning.

# 2. Website Audit Checklist

Use for public website review only.

Allowed:
- browser review;
- mobile check;
- PageSpeed/Lighthouse;
- public HTML/meta review;
- sitemap/robots check;
- contact form visual check;
- UX/SEO/content review.

Not allowed without permission:
- admin login attempts;
- password testing;
- vulnerability scanning;
- aggressive crawling;
- security testing;
- scraping personal data;
- automated mass outreach.

## Audit fields

For each lead, check:

- Website opens: yes/no
- HTTPS: yes/no
- Mobile version: good / medium / weak
- Clear offer: yes/no
- Clear CTA: yes/no
- Contact form: yes/no
- Contact page: yes/no
- Email visible: yes/no
- Phone visible: yes/no
- Trust signals: yes/no
- Legal notice / Impressum: yes/no
- Privacy policy: yes/no
- Meta title: good / weak / missing
- Meta description: good / weak / missing
- Main improvement opportunity:
- Suggested SmartFlow offer:
- Priority: low / medium / high

# 4. Proposal Checklist

Before sending a proposal, confirm:

- client problem is clear;
- recommended offer is selected;
- price range is justified;
- timeline is realistic;
- what is included is clear;
- what is not included is clear;
- client responsibilities are listed;
- next step is simple.

Proposal structure:

1. Context
2. Observed problem
3. Recommended solution
4. Deliverables
5. Timeline
6. Price
7. Conditions / assumptions
8. Next step

# 6. Outreach Rules

## Before sending

- Generate draft.
- Manually review.
- Shorten if needed.
- Remove technical wording.
- Check recipient.
- Check subject.
- Send manually.

## After sending

Only after real manual sending:

- mark lead as Contacted;
- verify contacted_at;
- keep note if needed.

## Do not

- do not mark Contacted before sending;
- do not send bulk emails;
- do not send to all Batch 001 at once;
- do not contact very large companies without better manual personalization.
