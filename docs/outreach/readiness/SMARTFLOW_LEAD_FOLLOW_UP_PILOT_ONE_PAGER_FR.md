# SmartFlow Lead & Follow-up Pilot

Document type: French client-facing one-pager  
Status: Draft  
Audience: Manual review before any client-facing use  
Related roadmap step: Step 2 — French One-Page Pilot Offer

---

## 1. Problème

Dans beaucoup d’entreprises, les demandes entrantes arrivent par plusieurs canaux : formulaire de contact, email, téléphone, site web ou demande directe.

Même lorsque le volume n’est pas très élevé, il peut être difficile de garder une vue claire sur :

- le type de demande ;
- le niveau d’urgence ;
- les informations manquantes ;
- la prochaine action à effectuer ;
- le suivi à ne pas oublier.

L’objectif n’est pas de remplacer l’équipe existante.  
L’objectif est de rendre le suivi plus clair, plus structuré et plus facile à vérifier manuellement.

---

## 2. Solution

SmartFlow Suisse propose un petit pilote Lead & Follow

1. une demande entrante est structurée dans un format clair ;
2. l’IA prépare un résumé interne ;
3. la demande est classée par type et priorité apparente ;
4. une prochaine action est proposée ;
5. une personne vérifie avant toute communication externe.

Le pilote reste volontairement limité, contrôlé et manuel au départ.

Il ne s’agit pas d’un système d’envoi automatique d’emails.  
Il ne s’agit pas d’un remplacement du processus interne de l’entreprise.  
Il s’agit d’une couche de support opérationnel pour mieux préparer le suivi.

---

## 3. Ce qui est inclus

Le pilote peut inclure :

- une structure simple pour centraliser les demandes entrantes ;
- un résumé IA de chaque demande ;
- une classification par type de demande ;
- une indication de priorité apparente ;
- une proposition de prochaine action ;
- une note interne de suivi ;
- une vérification humaine avant toute réponse au client ;
- une courte documentation du fonctionnement ;
- une session de revue après le premier test.

Le périmètre exact est confirmé avant le démarrage.

---

## 4. Déroulement

Déroulement recommandé :

1. Revue rapide du parcours de contact existant.
2. Définition du périmètre pilote.
3. Préparation d’un flux simple et contrôlé.
4. Test avec quelques demandes exemples ou demandes réelles validées.
5. Ajustement des règles de résumé, priorité et prochaine action.
6. Revue manuelle du résultat.
7. Décision sur la suite : garder le pilote, l’étendre ou l’arrêter.

Durée indicative :

- 2 à 5 jours ouvrables pour un périmètre simple ;
- plus long si le périmètre inclut plusieurs canaux, plusieurs équipes ou une intégration CRM plus avancée.

---

## 5. Sécurité

Le pilote est conçu avec une approche prudente.

Règles de sécurité :

- aucune campagne de masse ;
- aucun envoi automatique d’email externe ;
- aucune promesse de résultat commercial garanti ;
- aucune modification du processus interne sans validation ;
- aucune intégration avancée sans accord explicite ;
- revue humaine avant toute communication externe ;
- démarrage sur un périmètre limité.

Le but est de tester une amélioration concrète sans créer de risque opérationnel.

---

## 6. Prix pilote

Prix pilote indicatif :

- 1,200–1,500 CHF pour un périmètre simple autour des demandes entrantes et du suivi manuel ;
- le prix final dépend du périmètre confirmé, des outils existants et du niveau d’intégration nécessaire.

Pour une intervention plus limitée, par exemple une amélioration simple du parcours de contact ou une analyse courte, un budget inférieur peut être proposé séparément.

Pour un système plus avancé avec automatisation complète, CRM ou plusieurs flux internes, le prix doit être confirmé après compréhension du fonctionnement existant.

---

## 7. Prochaine étape

Prochaine étape proposée :

1. revoir ensemble le parcours de contact actuel ;
2. choisir un périmètre pilote limité ;
3. confirmer les règles de sécurité ;
4. préparer un premier test contrôlé ;
5. décider ensuite si le pilote mérite d’être étendu.

Le pilote peut être arrêté après la première phase si la valeur n’est pas suffisante.
